# Отчёт по лабораторной работе №5

---

## **Тема**

**Простое веб-приложение. Работа с API (Продолжение)**

---

## **Цель работы**

Целью данной лабораторной работы также является взаимодействие с внешним Api. Но если в прошлой лабораторной работе мы это делали с помощью **XMLHttpRequest**, то в этот раз мы будем пользоваться более современным "инструментом" - **fetch**.

В ходе выполнения работы, нам предстоит ознакомиться с небольшим, полезным теоретическим материалом, с кодом реализации простого взаимодействия с внешним Api, с получением данных и выводом их в интерфейс пользователя, а также нам предстоит выполнить задания по варианту.

---

## **Задание**

1. Изучить работу Call Stack и Event Loop.  
2. Разобраться с объектом Promise и его методами.  
3. Освоить использование Fetch API для HTTP-запросов.  
4. Изучить принципы DRY, KISS и SOLID.  
5. Ознакомиться с возможностями Web Storage API.  

---

## **Ход выполнения работы**

1. Изучен механизм **Call Stack** как структуры выполнения синхронного JavaScript-кода.  
2. Рассмотрена работа **Event Loop**, который управляет выполнением асинхронных задач и очередей событий.  
3. Изучены **Promise** как способ работы с асинхронными операциями и их состояния: `pending`, `fulfilled`, `rejected`.  
4. Рассмотрены методы `then`, `catch`, `finally`, а также статические методы `Promise.all`, `Promise.race`, `Promise.any`, `Promise.allSettled`.  
5. Освоен **Fetch API** для выполнения HTTP-запросов и обработки ответов сервера.  
6. Изучены принципы проектирования:
   - **DRY** — недопущение дублирования логики,
   - **KISS** — простота и понятность решений,
   - **SOLID** — набор принципов для построения масштабируемого и поддерживаемого кода.
7. Рассмотрены возможности **Web Storage API**: `localStorage` и `sessionStorage`, их назначение и методы.

---

## **Результат**

Получены теоретические и практические знания об асинхронной модели выполнения JavaScript, работе с HTTP-запросами, принципах архитектуры кода и клиентских хранилищах данных браузера.

---

## **Вывод**

В ходе лабораторной работы были изучены ключевые механизмы асинхронного JavaScript, что позволяет более эффективно работать с сетевыми запросами, событиями и состояниями приложения. Освоенные принципы проектирования и Web Storage API способствуют созданию более надёжных и поддерживаемых веб-приложений.
